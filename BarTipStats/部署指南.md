## 第 1 步：把代码推送到 GitHub

### 1.1 创建 GitHub 仓库

打开浏览器，登录 GitHub → 点右上角 `+` → `New repository`：

```
Repository name: BarTipStats
✅ Public（或 Private 都可以）
❌ 不要勾选 "Add a README"
✅ .gitignore: Swift
```

点 **Create repository**。

### 1.2 在 Windows 上推送代码

打开你的 VS Code（或终端），在项目目录执行：

```bash
cd d:\download\98tips\BarTipStats
git init
git add .
git commit -m "🎉 初始提交 - 酒吧小费统计 App"
git branch -M main
git remote add origin https://github.com/你的用户名/BarTipStats.git
git push -u origin main
```

> 💡 如果没装 Git：https://git-scm.com/download/win

## 第 2 步：GitHub 云端编译（自动执行）

代码推上去后，**GitHub Actions 会自动触发编译**。

### 查看编译进度

1. 打开你的 GitHub 仓库页面
2. 点击顶部 **Actions** 标签
3. 你会看到一个 `Build BarTipStats` 正在运行（黄色 ●）
4. 等它变成绿色 ✅（大约 5-10 分钟）

### 下载编译产物（.ipa 文件）

1. 点进这个绿色的 workflow 记录
2. 往下翻到 **Artifacts** 区域
3. 点击 **BarTipStats.ipa** 下载
4. 解压得到 `BarTipStats.ipa` 文件

> ⚠️ **如果编译失败**，通常是 Bundle ID 重复。到 `.github/workflows/build.yml` 里把 `BUNDLE_ID` 改成你能识别的名字再试一次。

---

## 第 3 步：用 Sideloadly 安装到 iPhone

### 3.1 下载安装 Sideloadly

去 https://sideloadly.io 下载 Windows 版并安装。

这是 Windows 上最流行的 IPA 安装工具，**免费、无需越狱**。

### 3.2 连接 iPhone

1. 用 USB 数据线把 iPhone 连到电脑
2. iPhone 上弹出「要信任此电脑吗？」→ 点 **信任** → 输入锁屏密码
3. 打开 **Sideloadly**

### 3.3 安装 IPA

在 Sideloadly 界面：

```
① iPhone → 自动识别你的设备
② IPA 文件 → 点 "IPA" 按钮 → 选择下载的 BarTipStats.ipa
③ Apple ID → 输入你的 Apple ID 邮箱
④ 点 "Start" 开始安装
```

它会让你输入 Apple ID 密码（或者用 **App-Specific Password** 更安全）。

### 3.4 信任开发者证书

安装完成后 iPhone 上会多一个 App 图标，但可能打不开：

```
iPhone → 设置 → 通用 → VPN与设备管理
→ 点击你的 Apple ID → 点「信任」
```

### 3.5 打开 App 🎉

回到桌面，找到 **"小费统计"** App，打开它！

## 常见问题

### ❌ Sideloadly 报错 "Anisette data"

安装最新版 **iTunes** 和 **Apple Application Support**（64位）：

1. 从 Apple 官网下载 iTunes（Windows 版）
2. 或者直接下载 Apple 应用支持：搜索 "AppleApplicationSupport64.msi"

### ❌ "Failed to generate an application identifier"

解决方法（任选一个）：
- 换一个 **Apple ID** 试试
- 去 https://appleid.apple.com 登录 → 安全 → 生成**App专用密码** → 用这个密码在 Sideloadly 登录

### ❌ App 安装后闪退 / 白屏

- 确认你的 iPhone 系统是 **iOS 17.0 以上**
- 删除 App 重新用 Sideloadly 装一次
- 重启 iPhone 再试

### ❌ GitHub Actions 编译报错

1. 去 GitHub 仓库 → Actions → 点进失败的记录
2. 查看红色错误日志
3. 最可能的原因：**Bundle ID 重复** → 去项目根目录 `.github/workflows/build.yml` 里修改 `BUNDLE_ID`
4. 改完后重新 commit 推送

## 备用方案：Codemagic（更简单）

如果觉得 GitHub Actions + Sideloadly 两步太麻烦，试试 **Codemagic** — 它一步到位：

1. 注册 https://codemagic.io （用 GitHub 账号登录）
2. 点 **"Add Application"** → 选择你的 GitHub 仓库
3. 点 **"Start new build"** → 选 iPhone 平台
4. 编译完成后 → 点 **"Install"** → 扫码自动安装到 iPhone

> 免费额度：每月 500 分钟构建时间，完全够用。
